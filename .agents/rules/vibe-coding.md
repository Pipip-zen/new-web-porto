Vibe coding rules:

- Start with the thinnest end-to-end slice that can be seen or run.
- Bias toward shipping over architecture speculation.
- Keep one bold idea per pass; do not redesign everything at once.
- Preserve momentum: if blocked, reduce scope, do not stall.
- Prefer edits that are easy to verify visually or with a quick command.
- For frontend work, make intentional aesthetic choices; avoid generic layouts.
- Before done, check mobile layout, empty states, focus states, and loading/error behavior.
- If a request risks a large refactor, split it into a visible first pass and a cleanup pass.
